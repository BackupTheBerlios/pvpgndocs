Introduction

What is PvPGN?

PvPGN stands for Player -vs- Player Gaming Network and is a complete emulation of Blizzard's Battle.net. Now that emulation of Battle.net is complete (this was done based on Bnetd's code), PvPGN is being recoded from scratch in cpp so support for additional games and software can be added and included in the future with open source coding.

Installation/Setup Questions

How do I setup a PvPGN server in Windows?

Download the latest version of PvPGN from here. The setup, described below, will allow you to play Warcraft II Battle Net Edition, Warcraft III Reign of Chaos, Starcraft, Starcraft Brood War & Diablo. You will be unable to play Diablo II or Diablo II Lord of Destruction since they require d2cs, d2dbs and / or d2gs / D2Closed which is not discussed in this section.

I suggest getting your PvPGN server to work first before pursuing the d2* programs since you will be unable to use those programs unless PvPGN is setup & working properly. Also, make sure that a Warcraft III loader is being used to connect to non-official Battle.net servers, when connecting with Warcraft III.

  1. To install PvPGN:

    • Simply run the exe file by double/single (depending how your system is setup) clicking on it.

    • Follow the instructions in the installer & click on finish to exit the installer when it's complete.

    If you are using Windows XP, 2000, ME, or NT, you might want to consider installing PvPGN as a service instead, though this is not required.

    To install PvPGN as a service, run the install program using the "-s install" switch. Example:

    • c:\pvpgn1.4.4.exe -s install

      Setup will say that it is "Installing as a Service".

      To start the PvPGN service, click on:

    • Start
    • Settings
    • Control Panel
    • Administrative Tools
    • Services
    • Select the PvPGN service & click on the play like looking button to start the service

      If your experiencing problems, the exe file that you want to use as the service may need to be renamed to PvPGNEliteService.exe .

    If you want to uninstall the PvPGN service, run the install program again using the "-s uninstall" switch. Example:

    • c:\pvpgn1.4.4.exe -s uninstall


  2. Now that PvPGN is installed. We need to configure PvPGN correctly before we can use it. Go to the location where you installed PvPGN. All of your configuration files, such as the bnetd.conf & log files are located there in sub-directories.

    The default installation path is:

    • c:\program files\the player -vs- player gaming network\pvpgn


  3. Go to c:\program files\the player -vs- player gaming network\pvpgn\etc & edit the bnetd.conf file with a text editor such as Notepad or Wordpad.

    Their really are only 4 things that must be changed in the bnetd.conf file to get PvPGN running. For now, leave all other settings alone, PvPGN should work just fine without changing any other settings. Only change the settings specified below:

    In the bnetd.conf file, look for the "Storage section" section. Change the storage_path to either the desired storage type, either the file: or sql: mode. Each are illustrated below:

    • storage_path = file:c:\program files\the player -vs- player gaming network\pvpgn\users
      Stores user account data via flat files in the directory c:\program files\the player -vs- player gaming network\pvpgn\users. No sql database is used in this mode.

      or

    • storage_path = sql:mode=mysql;host=127.0.0.1;name=PVPGN;user=pvpgn;pass=pvpgnrocks
      Stores user account data via an sql database. This is more ideal for lots of users and/or live stats. This setting must also be in one single line (as illustrated above). For a break down of each sql setting, see How do I setup SQL support for PvPGN in Windows?.

    Scroll down, look for the "Message Logs" section. Change the loglevel to the following:

    • loglevels  =  fatal,error,warn,info,trace,debug
      This tells PvPGN to log all possible errors including initialization, MySQL & connection errors.

    Scroll down until you get to the "Server Network Info" section. Change the servaddrs to the following:

    • servaddrs  =  "0.0.0.0:6112"
      This will tell PvPGN to listen on port 6112 for both the TCP & UDP protocols, for all IP addresses on your computer.

    For the last line that needs to be changed in the bnetd.conf file, simply scroll down a few lines and change the w3routeaddr, which is also under the "Server Network Info" section, to the following:

    • w3routeaddr  =  "X.X.X.X:6200"
      Replace X.X.X.X with your IP address. This must be a specific IP address or a domain name such as www.pvpgn.org, 0.0.0.0:6200 won't work. This tells the PvPGN Warcraft III router to listen on port 6200 for the TCP protocol. This will route Warcraft III data to PvPGN.

    Scroll down once again & until you get to the w3routeshow line. Enable & change this only if your using NAT, behind a router, gateway, etc. If your not sure, just leave this alone.

    Try using w3routeaddr first, if your having problems connecting with Warcraft III, then enable w3routeshow.

    • W3routeshow is disabled by default & should be left that way unless your experiencing problems.

    • #w3routeshow  =  "127.0.0.1:6200"
      This tells PvPGN to listen on port 6200 for the TCP protocol, on localhost (127.0.0.1), on your computer. If you enable this, change this to your internet IP address or domain name, followed by the colon and port number.

    Save & close your text editor.


  4. Now, open ports 6112 & 6200 if your using a firewall. If your behind a router, forward ports 6112 & 6200 to your computer or register the internal IP address of the computer running PvPGN as a DMZ in the router. If your using Windows XP, it has a firewall that is installed by default. Make sure to allow incoming & outgoing access to ports 6112 & 6200, if the firewall is enabled, or you will be unable to connect to your PvPGN server.


  5. Now your ready to start PvPGN.

    On your taskbar, click on:
    • Start
    • Programs
    • Player -vs- Player Gaming Network
    • PvPGN

After starting PvPGN, open your bnetd log file. It should look something like this:

How do I setup a PvPGN server on *nix based systems?

First, download the source. You can download the latest version here. Then, proceed to install PvPGN.
  1. Installing PvPGN on *nix based systems...has many more options than the Windows version. Read this section thoroughly before doing anything as you may wish to use different options based on your needs or system setup. The tar file must file be uncompressed first. Login as root, go to the directory where you downloaded PvPGN & type something like this:

    • tar xvzf pvpgn.tar.gz
      This tells tar to create a directory using the tar file name & uncompress the files in the tar file to that directory. Make sure you change pvpgn.tar.gz to the tar file name your going to uncompress.

    • cd pvpgn/src
      This changes us to the pvpgn directory where you uncompressed the tar file, with the sub-directory src, which is where the PvPGN source is located.

      Now we need to configure the installer before we can compile. Their are several different switches for this depending whether you want to install PvPGN with MySQL support, change the default installation location, etc. Only the most common switches are used below all though their are more switches than illustrated below.

    • ./configure
      This will configure PvPGN with the default install location & without MySQL. User accounts will be stored via flat files. PvPGN's files will be installed in /usr/local in subdirectories.

    • ./configure --with-mysql
      Using the --with-mysql switch will configure PvPGN using the MySQL database to store it's user accounts & stats. This is a good idea if you plan on having a lot of user accounts. Requires MySQL to be installed & running.

    • ./configure --prefix="/home/www/bnetd"
      By default, PvPGN will install to /usr/local. If you want it to install to a different directory, change the --prefix path to where ever you want to install PvPGN. In this example, PvPGN would install to /home/www/bnetd.

    • ./configure --with-mysql --prefix="/home/www/bnetd"
      You can also use the switches together. Such as the above example would install MySQL support and install PvPGN in the /home/www/bnetd directory

      Now that installer for PvPGN is configured. We can compile it using standard *nix commands. Type this at the bash prompt:

    • make
      This compiles PvPGN based on the setting from the configure script.

    • make install
      Make install copies all the appropriate files to their locations. Otherwise, you'd have to manually copy the compiled files to their proper locations or simple run PvPGN from where you compiled it.


  2. Now that PvPGN is installed, we need to configure PvPGN correctly before we can use it. Go to the location where PvPGN is installed. All of your configuration files, such as the bnetd.conf & log files are located there in sub-directories.

    The default installation path for *nix based systems is:

    • /usr/local


  3. Go to the etc sub-directory & edit the bnetd.conf file with a text editor such as emacs, xemacs, pico, vi, etc.

      or

      Stores user account data via an sql database. This is more ideal for lots of users and/or live stats. This setting must also be in one single line (as illustrated above). For a break down of each sql setting, see How do I setup SQL support for PvPGN on *nix based systems?.

    Scroll down, look for the "Message Logs" section. Change the loglevel to the following:

    • loglevels  =  fatal,error,warn,info,trace,debug
      This tells PvPGN to log all possible errors including initialization, MySQL & connection errors. It might be a good idea, after PvPGN is up & running, to scale these setting back to something like fatal,warn. This will help log unwanted jargon & make the log easier to read.

    Scroll down until you get to the "Server Network Info" section. Change the servaddrs to the following:

    • servaddrs  =  "0.0.0.0:6112"
      This tell's PvPGN to listen on port 6112 for both the TCP & UDP protocols, for all IP addresses on your computer.

    For the last line that needs to be changed in the bnetd.conf file, simply scroll down a few lines and change the w3routeaddr, which is also under the "Server Network Info" section, to the following:

    • w3routeaddr  =  "X.X.X.X:6200"

    Scroll down once again & until you get to the w3routeshow line. In PvPGN version 1.1.6x & newer, a w3routeshow option is available. Enable & change this only if your using NAT, behind a router, gateway, etc. If your not sure, just leave this alone.

How do I setup SQL support for PvPGN in Windows?

Make sure that both SQL & PvPGN are installed. Close PvPGN if it is running, then:


  1. Edit the bnetd.conf file, located in the c:\program files\the player -vs- player gaming network\pvpgn\etc directory by default. Scroll down to the bottom of the file to the "Storage section" section. In most cases, just leave everything the way it already is by default. Make sure this command is one line (as illustrated below).

    storage_path = sql:mode=mysql;host=127.0.0.1;name=PVPGN;user=pvpgn;pass=pvpgnrocks

    Settings broken down...
    • mode  =  mysql
      The type of sql database PvPGN will be connecting to. Options could be mysql, pgsql and so forth.

    • host  =  localhost
      The computer sql is installed on & where the database for PvPGN will be kept. 90% of the time, localhost will work just fine. This tells PvPGN to connect to the same computer it's running on for the sql database. If you are running sql on a different computer other than the computer PvPGN is running on, change this to the IP address of the remote computer that has sql installed on it.

    • name  =  PVPGN
      The database name that PvPGN will use. In most cases, just leave this alone.

    • user  =  pvpgn
      The user name that the PvPGN will use to log into the database with. In most cases, just leave this alone.

    • pass  =  pvpgnrocks
      The password that the PvPGN will use to log into database with. In most cases, just leave this alone.

    Save & close your text editor, then start PvPGN.

How do I setup D2Closed, what's it for & where can I get it?

D2Closed is the Windows version of the D2CS & D2DBS programs which are the programs required to host both open & closed Diablo II games. Running PvPGN & D2Closed, without d2gs, will only support Diablo II open player support. What's an Open Server? Click here to find out.

Before making any configuration changes, make sure that PvPGN works ok on it's own before attempting to run D2Closed. D2Closed will attempt to connect & create a link between it and PvPGN. This is why it is so important to have PvPGN working before attempting to run D2Closed.

Also, the D2Closed server requires the Lord of Destruction expansion pack files, so make sure it's installed. If you try to run D2Closed in Classic mode, the service will report a "Server Mutex" error and die.

Lets start the installation process:


  1. Download & install the D2Closed Realm Server.


  2. Go to the path where D2Closed is installed, in the conf sub-directory & edit the d2closed.conf file.


  3. Look for the "Server Major Settings" section. Scroll down to the d2cs_servaddrs field, make it look something like this:

      This tells the D2CS portion of D2Closed to listen on port 6113, using the TCP protocol, on all IP addresses.


    Scroll down to the d2dbs_servaddrs field & make it look like this:

    • d2dbs_servaddrs  =  0.0.0.0:6114
      This tells the D2DBS portion of D2Closed to listen on port 6114, using the TCP protocol, on all IP addresses.

      • servaddrs  =  0.0.0.0:6113
        This tells D2CS to listen on port 6113, using the TCP protocol, on all IP addresses.

      • loglevels  =  fatal,error,warn,info,trace,debug

        This setup would disable tracking info.

        This setup would send tracking info to what ever IP address you define on port 6114 (replace X.X.X.X with your internet IP address or what ever IP address you want to send the tracking info to).     How do I setup bntrackd & what's it for?

    The Bntrackd program receives stats information from bnetd based servers, interprets the data & dumps it into a text file. From their, the perl script process.pl, located in the source under the scripts directory, converts the text dump to an html page. Like the page seen at the PvPGN Server List. Which intern, produces a stats page ranging from one to multiple game servers from all over the world. It also includes valuable information such as game server uptime, users online, games in progress, etc.

    If you want to send tracking info from bnetd to yourself, you will need to use bntrackd & have UDP port 6114 open.

    1. In *nix based systems, go to where you installed PvPGN and go to the sbin directory. Run:

      • ./bntrackd -o /usr/local/bin/bntrackd.txt -l /var/log/bnetd/bntrackd.log

      The -o (which stands for output) tells bntrackd to dump it's incoming info to /usr/local/bin/bntrackd.txt and the -l (which stands for log) logs any errors to /var/log/bnetd/bntrackd.log


    2. Make sure trackaddrs in the bnetd.conf file is set to either localhost, your internet IP address, or your internal IP address. Then, copy the process.pl from the source/scripts directory to a desirable path, make it executable and make sure the shebang path (example: #!/usr/bin/perl) is correct for the system your running on. Edit the process.pl file, change 3 lines:

      • $infile  =  "/usr/local/bin/bntrackd.txt";
        This is the path you told bntrackd to dump it's data to.

      • $outfile  =  "/var/www/html/bnetdlist.html";
        This is the path & name, where you want the html file to be produced. If you want this accessible by others, it must be in a location readable from the internet.

      • $tmpfile  =  "/tmp/bntrackd.tmp";
        This is the path where process.pl will place it's temporary file when running.

      • type perl process.pl

    If all goes well, you should have a bnetdlist.html page in /var/www/html. Then, setup a cron script to run the process.pl every few minutes or you could modify the perl script so it can be accessible directly from the internet and dump it's data directly to the screen rather than to an html file. It also might be a good idea to add step 1 to your rc.local file, so bntrackd will load at boot time automatically.

    General Questions

    What is a Diablo II Open Server & Diablo II Closed Server?

    The answer to this question is overall simple.

    1. A Diablo II Open Server means that your character is not stored on PvPGN like on the real Battle.net. When you play on Battle.net, your player is stored on their server. In open Battle.net game play, your character is stored on your local computer instead. This requires D2CS & D2DBS to be running in addition to PvPGN on the server side. It has several advantages & disadvantages:

      Advantages:
      • Single player character can be used to play on Battle.net.
      • Backups can be made of your character in case your character file gets hosed or lost.

      Disadvantages:
      • Players can easily cheat.
      • Trainers maybe used through out game play undetected.
      • Admin has very little to no control over player's characters.
      • Players can use hacks on their characters, weapons, armor, etc. to make them extremely powerful.

      To play on an Open Battle.net Server, simply:
      • Start Diablo II or Diablo II LOD.
      • Select the Gateway of the machine you want to connect to.
      • Click on the Other Multiplayer button.
      • Click on the Open Battle.net button.


    2. A Diablo II Closed Server means that your character is stored on PvPGN just like the real Battle.net. This requires D2CS, D2DBS & D2GS to be running in addition to PvPGN on the server side. It has several advantages & disadvantages as well:

      Advantages:
      • Hacks & trainers are much easier to detect.
      • Players have a much harder time trying to cheat.
      • The admin of the server has more control over player's characters.

      Disadvantages:
      • Single player characters cannot be used.
      • Backups can't be made of your character by the player.
      • Slow servers or bandwidth can cause lagging or additional wait time.
      • Player is at the mercy of the admin, should the admin wish to delete your character.
      • If you don't log in frequently, your account could be deleted, along with your character.

      To play on an Closed Battle.net Server, simply:
      • Start Diablo II or Diablo II LOD.
      • Select the Gateway of the machine you want to connect to.
      • Click on the Battle.net button.


    3. You may run both an open & closed server at the same time. The player, in such a situation, would then choose which game play they would like to use (open or closed).

Once that user has administrative privileges, they can give other players the same privileges, should he or she wish to, within the Battle.net console. If you already have a user with administrative privileges & want to give another user the same privileges, skip to step 3 for further information. If you installed PvPGN without MySQL support, it means you are using the flat files to store players account information & stats. The user accounts are typically named by the users name. The PVPGN database is not located here.

    1. If you installed PvPGN without MySQL support: :

      • Shutdown PvPGN.

      • Find the name of the user you want to give administrative privileges in the path where you installed PvPGN, under the var/users sub-directory.

      • Edit the file in a text editor & add the following lines to the beginning of the user account file. Make sure they are the first lines in user account files or it wont work:

        • "BNET\\auth\\admin"="true"

        • "BNET\\auth\\operator"="true"

      • Save & close your text editor.

      • Start PvPGN.


    2. If you installed PvPGN with MySQL support:

      • Shutdown PvPGN.

      • Edit the PVPGN database. phpMyAdmin is a good program to use to edit MySQL databases.

      • Go to the BNET table in the database.

      • Find the user account you want to give administrative privileges.

      • Change the auth column from false to true.

      • Apply the changes.

      • Start PvPGN.


    3. Now that you have at least one user with administrative privileges, log in as that user & type the following to give administrative privileges to another user. Only give this privilege to users you trust. This privilege is NOT meant for everybody, just those that run or manage your PvPGN server. All users are NOT given administrative privileges by default:

      • /makeadmin jack
        This example would give the user jack administrative privileges.

      To remove administrative privileges, at the console type:

      • /demoteadmin jack
        This example would remove the user jack administrative privileges.

      This works instantly when PvPGN is setup using MySQL. If your running in file mode, PvPGN will need to be restarted for the changes to take effect.

    What's the difference between the Standard & Elite version of PvPGN?

    This is unreal and will most likely be removed in the next FAQ update. If your reading this section, it would be best to upgrade to the latest version of PvPGN, since their is know longer an "elite" version of PvPGN.

    The "standard" & "elite" version are for Windows only. For *nix based systems, just configure PvPGN properly before compiling it. Here are some pros & cons to each setup:


    1. Standard version:

      • User accounts are stored via flat files under the var\users directory.
      • SQL support is unavailable in this version.
      • No profiles, ranking, web stats support.
      • May be slow starting/restarting PvPGN with lots of user accounts.


    2. Elite version:

      • User accounts are stored via an SQL database.
      • Since elite stores it's data via SQL, profiles, rankings & web stats are available.
      • PvPGN may be run as a service instead of as an application.
      • With lots of users, this version starts/restarts faster.
      • Making changes to user accounts are easier.
      • If something happens to the database, all user accounts are lost, unless it's backed up.
      • The prior version's database might be incompatible with new up-coming versions of PvPGN. Which, in short, means loss of user accounts including their stats. Example: MySQL database from PvPGN 1.1.5 may be incompatible or lack features for use with PvPGN 1.1.7.

    Trouble Shooting

    When I look in my log file after starting PvPGN, it says "main: failed to initialize network (exiting)".

    Make sure you edited the bnetd.conf file correctly. Their are two possibilities as to what causes this:
    1. The servaddrs in the bnetd.conf is set incorrectly, make the servaddrs look like this:

      • servaddrs  =  "0.0.0.0:6112"
        This will tell PvPGN to listen on all IP addresses for port 6112.

      or if you want PvPGN to listen only on a specific IP address, make the servaddrs look like this:

      • servaddrs  =  "X.X.X.X:6112"
        Replace X.X.X.X with your internet IP address.
        If your behind a router, replace X.X.X.X with your internal IP address.


    2. The w3routeaddr in the bnetd.conf is set incorrectly, make the w3routeaddr look like this:

      • w3routeaddr  =  "X.X.X.X:6200"
        Replace X.X.X.X with your internet IP address.
        If your behind a router, replace X.X.X.X with your internal IP address.


    Example exert from the bnetd.conf file:
    ############################################################################# #
    # Server network info #
    #----------------------------------------------------------------------------------------------------- #
    # Set this option to true to allow TCP to detect and close stale #
    # connections. #
    use_keepalive  =  false
    # #
    # This is a comma delimited list of hostnames that the server should #
    # listen on. It might be useful to make an internal-only server on a #
    # gateway machine for example. If the list is not set or if it has a #
    # entry with no host component, the server will bind to that port on all #
    # interfaces. #
    servaddrs  =  "26.135.61.243:6112"
    # #
    # Don't change these unless you really need to! You will need to run a proxy #
    # or modify the clients. Also note that these will not change when simply #
    # sending a HUP signal to the server; they are only read on startup. #
    # #
    # This is the port the server send the UDP test packets to by default. #
    # Setting it to zero makes the server use the same port as the TCP connection #
    # comes from. Newer clients can override this setting on a per connection #
    # basis. #
    #udptest_port  =  6112
    # #
    # added by zap-zero: #
    # W3 Play Game router address. Just put your server address in here. #
    # It's needed to tell clients where to connect to init anonymous games. #
    # Change it, or clients will get "Error handling the request"! #
    w3routeaddr  =  "26.135.61.243:6200"
    # #
    ############################################################################# #

    I'm having problems connecting to my PvPGN server.

    This is typically caused by a firewall or router that is not setup properly:
    1. Make sure you have at least port 6112 open for the TCP & UDP protocols on both the server side and client side if your behind a firewall.

    2. If your behind a router, make sure your router it is setup to forward the appropriate ports to your server, else register it in the router as a DMZ. Be aware that if you register your computer as a DMZ in your router, it will be completely exposed to the internet, as if it were directly connected to the internet, without protection from the router. In this setup, you will have no protection from intruders and some sort of firewall would be recommended.

      Here are some good links on how to setup your router:
      • Linksys Router Setup for Warcraft 3
      • Battle.net Router/Firewall Configuration (Mirror)

    3. If your trying to connect using Warcraft III, make sure you are using the binary (exe) loader or the AX loader. Also, make sure your using the correct version loader for the version game your trying to connect with. Example - The loader for Warcraft III version 1.01 wont work with Warcraft III version 1.02.

    4. If your trying to connect using the same computer that your running the server on, you will probably get a connection error. This really depends on what OS your running on. In Windows XP, probably Windows 2000 as well, you can connect a client to a server on the same computer without any problems, however you will be unable to create games, just join them and PG/AT wont work either. This arrangement probably will not work on Windows 98 and earlier.

    5. If all else fails, check your log files. Most of the time, something in one of the config files are set incorrectly which is causing your connection problems. If you are still unable to connect to your PvPGN server, try logging on to irc.pvpgn.org, channel #pvpgn. Someone their might be able to assist you.
